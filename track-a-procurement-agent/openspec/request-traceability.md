# Request Traceability (Challenge 2)

This document manually traces each sample request in mock_data/requests.json through the four tool specs:
- budget-tool
- vendor-duplication-tool
- policy-compliance-tool
- risk-assessment-tool

## Trace Assumptions

1. All four tools are called for every request.
2. Final decision priority is: escalate > deny > approve.
3. POL-001 deny trigger from vendor duplication is applied when amount is above 25,000 and the request attempts a non-contracted vendor while active contracted alternatives exist.
4. Budget evaluation uses remaining budget values from mock_data/budgets.json.
5. REQ-015 is intentionally ambiguous in sample data and is called out explicitly.
